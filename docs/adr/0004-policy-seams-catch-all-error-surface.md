# Four catch-all policy seams on BaseNode are the error-handling surface

Developer-facing error handling (and node guarding generally) is four **catch-all policy seams** inherited by every *caller-capable* node type from `BaseNodeDef` — `before_node`, `after_node`, `on_node_error`, `on_callee_error` — with one uniform contract: *returning `None` is a no-op; returning a value makes it the output of the thing the seam guards* (skip / replace / recover / resolve follow from position). Consumers/observers have **no seams** (their body is entirely user-authored, so the lack-of-authorship rationale doesn't apply; `ConsumerContext` — including `ctx.fault` — is their whole surface, and observers never produce on the workflow's rails). Seams are chains (list, registration order, first non-`None` wins — specific handlers before catch-all converters). Type-specific error logic branches **inside** the handler body on the slotted `ErrorReport`; the client edge raises a single `NodeFaultError` carrying the same report. `gate()` is retired — `before_node` strictly supersedes it.

We chose catch-all seams over per-error-type registration because typed registration demands taxonomy knowledge before a developer can write their first handler and makes the most important handler (catch everything) the awkward one; every comparable framework (Google ADK `on_tool_error`/`on_model_error`, Spring Kafka `CommonErrorHandler`, NServiceBus recoverability policy) ships one catch-all seam per cross-section. The surface deliberately mirrors ADK 2.x for DX familiarity while fixing its verified irregularities: one return predicate (`is None`) everywhere; a raise inside `before_node`/`after_node` routes to `on_node_error` instead of bypassing the error seams; a raise inside `on_callee_error` is slot-scoped (the slot fails with the error chained — never a whole-invocation outcome mid-batch); and a deliberate `raise NodeFaultError(...)` bypasses `on_node_error` everywhere (the mint gesture must not be convertible into a success by an unrelated recovery handler).

## Considered options

- **Per-error-type handlers** (typed registration). Rejected as the canonical surface (too low-level day one); per-type sugar can be added later — the `x-calf-error-type` header already carries the key. (Fault deliveries carry no route header under ADR-0006, so the route registry is not, and cannot be, the error-dispatch plumbing.)
- **Errors-as-values in every continuation** (Result-branching). Rejected as the primary seam: it taxes every happy path with a branch, and the agent's continuation is the framework's own loop, so users would need subclassing to participate. It survives where it is correct: sibling faults inside a fan-out batch are values the aggregator examines.
- **No self-error seam** (earlier recommendation). Reversed: developers lack authorship inside agent/MCP-toolbox bodies (no user code can `try/except` the model call), so the edge needs `on_node_error`; it also seeds node-local retry (the successor retry issue) without round trips. The contract prevents swallowing: recovery must produce an output, `None` produces a fault — *nothing* is impossible.
- **error_type→exception-class registry at the client.** Rejected: every touchpoint presents the same slotted `ErrorReport`; a typed-exception surface only at the client would be the one inconsistent interface.
- **Subclass alias names** (`before_model` ≡ agent `before_node`). Rejected: in ADK those are different seams (per-LLM-request vs per-invocation), and an agent invocation mid-aggregation makes no model call — the alias would mislead. Names stay reserved for true inner seams if demand appears.

## Consequences

- Gates, the `Silent` action, `FailedToolCall` (terminal path), and `ToolExecutionError` are deleted — pre-1.0 hard break. The seam contract is total (substitute / decline with `None` / raise — the standard hook-host off-ramps; no silent-drop gesture exists), so a no-output outcome on a reply-owing delivery auto-faults (`calf.delivery.rejected`), closing #201 by construction.
- Fault deliveries are classified by the `x-calf-kind` header *before* route dispatch and carry no route header at all (ADR-0006), so they structurally cannot reach the `'*'` catch-all — by construction, not by rule (this replaces the earlier load-bearing "reserved fault route must never match `'*'`" consequence).
- No middleware-shaped second family is planned (the `calfkit-v1-design.md` G5/§12 sketch is not a commitment); transport-altitude cross-cutting lives in FastStream's own middlewares beneath calfkit. If a wrap-shaped family is ever proposed, it must wrap fault dispatch identically to normal dispatch and obey the seam totality rule (it may not swallow a reply-owing delivery without producing or raising).
